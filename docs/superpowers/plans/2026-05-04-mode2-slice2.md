# Mode 2 Slice 2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the four deferred Mode 2 items: per-call `system_prompt` override, MCP-spec cancellation via `notifications/cancelled`, SSE progress streaming, and approval forwarding via polling tools.

**Architecture:** Slice-2 changes are fully additive on top of slice-1. We extend `AgentLoop` with cancellation + extra system message, extend the MCP HTTP transport with progress streaming + cancel routing, and add a small `ApprovalQueue` + two polling tools for forward-approval. Three new CLI flags + one new value for `--mode2-approval`. No breaking changes to slice-1 callers.

**Tech Stack:** Rust 2024-edition workspace, `tokio` + `tokio_util::sync::CancellationToken`, `tokio::sync::oneshot` for approval waiters, manual HTTP/1.1 SSE on `tokio::io::AsyncWrite`, `async_trait`, `serde_json`. No new external dependencies — every primitive already lives in the workspace.

**Source spec:** [`docs/superpowers/specs/2026-05-04-mode2-slice2-design.md`](../specs/2026-05-04-mode2-slice2-design.md).

**Workflow note:** Run `cd rust && cargo test -p <crate> --no-fail-fast` after every implementation task. Run the full `scripts/check.sh` at the end of each feature group (after Tasks 1.x, 2.x, 3.x, 4.x). Each task ends with a commit; commit messages follow the format `mode2: <one-line subject>` to keep the slice-2 history scannable.

```rust
use serde_json::Value;
use tokio::sync::mpsc;

/// JSON-RPC request id as observed on the wire. Slice 2 uses this to
/// route `notifications/cancelled` back to the in-flight `helper_ask`
/// call that originally claimed the id.
#[derive(Debug, Clone, PartialEq, Eq, Hash)]
pub enum RpcId {
    Number(i64),
    String(String),
}

impl RpcId {
    /// Parse a JSON-RPC id from its on-wire form. Returns `None` for
    /// `null` or non-string-non-number ids; those are treated as
    /// "no id" (notifications).
    pub fn from_json(v: &Value) -> Option<Self> {
        match v {
            Value::Number(n) => n.as_i64().map(Self::Number),
            Value::String(s) => Some(Self::String(s.clone())),
            _ => None,
        }
    }

    /// Render back to a JSON value so a dispatcher can echo it onto an
    /// outbound response.
    pub fn to_json(&self) -> Value {
        match self {
            Self::Number(n) => Value::Number((*n).into()),
            Self::String(s) => Value::String(s.clone()),
        }
    }
}
```

//! In-process registry mapping JSON-RPC ids to per-call
//! [`CancellationToken`]s.
//!
//! Slice 2's `helper_ask` registers a token under the inbound
//! request id on entry and removes it on exit. The dispatcher's
//! `notifications/cancelled` handler looks up the id and fires the
//! token, which causes the corresponding `process_streamed_with_cancel`
//! to bail out promptly.

use std::collections::HashMap;
use std::sync::Mutex;

use tokio_util::sync::CancellationToken;
use zunel_tools::RpcId;

#[derive(Default)]
pub struct CancelRegistry {
    inner: Mutex<HashMap<RpcId, CancellationToken>>,
}

impl CancelRegistry {
    pub fn new() -> Self {
        Self::default()
    }

    /// Register a fresh token under `id`. Replaces any existing
    /// token for that id (caller is expected not to double-register
    /// the same id; the replace branch is here for paranoia rather
    /// than as a supported flow).
    pub fn register(&self, id: RpcId) -> CancellationToken {
        let token = CancellationToken::new();
        self.inner
            .lock()
            .expect("CancelRegistry lock poisoned")
            .insert(id, token.clone());
        token
    }

    /// Look up `id` and fire its token. Returns `true` when an entry
    /// was found and cancelled, `false` when no in-flight call had
    /// claimed the id (already completed, never registered, …).
    pub fn cancel(&self, id: &RpcId) -> bool {
        let mut guard = self.inner.lock().expect("CancelRegistry lock poisoned");
        match guard.remove(id) {
            Some(token) => {
                token.cancel();
                true
            }
            None => false,
        }
    }

    /// Drop the entry without cancelling. Called by the registering
    /// task on its happy-path exit so the table doesn't leak.
    pub fn unregister(&self, id: &RpcId) {
        self.inner
            .lock()
            .expect("CancelRegistry lock poisoned")
            .remove(id);
    }

    /// Test helper.
    #[cfg(test)]
    pub fn len(&self) -> usize {
        self.inner.lock().expect("CancelRegistry lock poisoned").len()
    }
}

/// RAII guard that registers an id on construction and unregisters
/// on drop, so happy-path callers never need to remember to clean
/// up. Cancelled calls fall through `unregister` cheaply.
pub struct CancelGuard<'a> {
    registry: &'a CancelRegistry,
    id: RpcId,
    pub token: CancellationToken,
}

impl<'a> CancelGuard<'a> {
    pub fn new(registry: &'a CancelRegistry, id: RpcId) -> Self {
        let token = registry.register(id.clone());
        Self {
            registry,
            id,
            token,
        }
    }
}

impl<'a> Drop for CancelGuard<'a> {
    fn drop(&mut self) {
        self.registry.unregister(&self.id);
    }
}

mod tests {

#[async_trait]
pub trait McpDispatcher: Send + Sync + 'static {
    async fn dispatch(&self, message: &Value, meta: &DispatchMeta) -> Option<Value>;

    /// Streaming variant. Default-implemented: delegates to
    /// [`McpDispatcher::dispatch`] and ignores the progress sink, so
    /// existing dispatchers (Mode 1, stdio) get correct behaviour for
    /// free. Mode 2's dispatcher overrides this to forward

    /// caller's responsibility.
    async fn dispatch_streaming(
        &self,
        message: &Value,
        meta: &DispatchMeta,
        _progress_sink: mpsc::Sender<Value>,
    ) -> Option<Value> {
        self.dispatch(message, meta).await
    }
}

```rust
/// Streaming SSE response writer used when the client opted in via
/// `_meta.progressToken`. Emits one chunked SSE `event: message`
/// frame per progress notification received from the dispatcher,
/// plus a final frame carrying the JSON-RPC result envelope.
async fn write_streaming_response<S>(
    stream: &mut S,
    session_id: &str,
    mut progress_rx: mpsc::Receiver<Value>,
    final_response: impl std::future::Future<Output = Option<Value>>,
) -> Result<()>
where
    S: AsyncWrite + Unpin,
{
    let header = format!(
        "HTTP/1.1 200 OK\r\n\
         Content-Type: text/event-stream\r\n\
         Cache-Control: no-cache, no-transform\r\n\
         Mcp-Session-Id: {}\r\n\
         Transfer-Encoding: chunked\r\n\
         Connection: close\r\n\r\n",
        session_id
    );
    stream.write_all(header.as_bytes()).await?;
    stream.flush().await?;

    tokio::pin!(final_response);

    loop {
        tokio::select! {
            biased;
            Some(progress) = progress_rx.recv() => {
                let body = format!(
                    "event: message\ndata: {}\n\n",
                    serde_json::to_string(&progress)?
                );
                write_chunk(stream, body.as_bytes()).await?;
                stream.flush().await?;
            }
            done = &mut final_response => {
                // Drain any progress events still queued behind the
                // dispatcher's response future.
                while let Ok(progress) = progress_rx.try_recv() {
                    let body = format!(
                        "event: message\ndata: {}\n\n",
                        serde_json::to_string(&progress)?
                    );
                    write_chunk(stream, body.as_bytes()).await?;
                }
                if let Some(value) = done {
                    let body = format!(
                        "event: message\ndata: {}\n\n",
                        serde_json::to_string(&value)?
                    );
                    write_chunk(stream, body.as_bytes()).await?;
                }
                write_chunk(stream, b"").await?;
                stream.flush().await?;
                let _ = stream.shutdown().await;
                return Ok(());
            }
        }
    }
}
```

//! Per-session pending-approval queue + oneshot waiter map.
//!
//! Slice 2's `forward` approval policy enqueues an
//! `ApprovalRequest` into a session-keyed queue and blocks the
//! requesting `ForwardingApprovalHandler` on a oneshot. The hub
//! polls via `helper_pending_approvals`, retrieves the queued items,
//! and submits answers via `helper_approve` which fires the oneshot.
//!
//! This module is transport-agnostic; the polling tools build on
//! top.

use std::collections::HashMap;
use std::sync::Mutex;
use std::time::SystemTime;

use serde_json::{json, Value};
use tokio::sync::oneshot;
use zunel_core::{ApprovalDecision, ApprovalRequest};

/// Per-session capacity. Once exceeded, `enqueue` fails; the helper
/// surfaces this as a structured error to the caller. 16 is a generous
/// upper bound for an interactive agent loop.
pub const PER_SESSION_CAPACITY: usize = 16;

#[derive(Debug, Clone)]
pub struct PendingApproval {
    pub request_id: String,
    pub tool_name: String,
    pub args: Value,
    pub description: String,
    pub scope: String,
    pub requested_at: SystemTime,
}

impl PendingApproval {
    pub fn to_json(&self) -> Value {
        json!({
            "request_id": self.request_id,
            "tool_name": self.tool_name,
            "args": self.args,
            "description": self.description,
            "scope": self.scope,
            "requested_at": humantime::format_rfc3339_seconds(self.requested_at).to_string(),
        })
    }
}

#[derive(Default)]
struct SessionQueue {
    pending: Vec<PendingApproval>,
    waiters: HashMap<String, oneshot::Sender<ApprovalDecision>>,
}

#[derive(Default)]
pub struct ApprovalQueue {
    inner: Mutex<HashMap<String, SessionQueue>>,
    next_seq: std::sync::atomic::AtomicU64,
}

impl ApprovalQueue {
    pub fn new() -> Self {
        Self::default()
    }

    /// Enqueue a request. Returns the assigned `request_id` and a
    /// receiver the forwarding handler awaits. Errors with a String
    /// when the per-session cap is reached.
    pub fn enqueue(
        &self,
        session_id: &str,
        req: ApprovalRequest,
    ) -> Result<(String, oneshot::Receiver<ApprovalDecision>), String> {
        let request_id = format!(
            "ap-{}",
            self.next_seq
                .fetch_add(1, std::sync::atomic::Ordering::Relaxed)
        );
        let mut guard = self.inner.lock().expect("ApprovalQueue lock poisoned");
        let entry = guard.entry(session_id.to_string()).or_default();
        if entry.pending.len() >= PER_SESSION_CAPACITY {
            return Err(format!(
                "approval queue for session {session_id} is at capacity ({PER_SESSION_CAPACITY})"
            ));
        }
        let pending = PendingApproval {
            request_id: request_id.clone(),
            tool_name: req.tool_name,
            args: req.args,
            description: req.description,
            scope: format!("{:?}", req.scope),
            requested_at: SystemTime::now(),
        };
        let (tx, rx) = oneshot::channel();
        entry.pending.push(pending);
        entry.waiters.insert(request_id.clone(), tx);
        Ok((request_id, rx))
    }

    /// Snapshot the pending entries for a session without removing
    /// them. The hub calls this from `helper_pending_approvals`.
    pub fn pending(&self, session_id: &str) -> Vec<PendingApproval> {
        self.inner
            .lock()
            .expect("ApprovalQueue lock poisoned")
            .get(session_id)
            .map(|q| q.pending.clone())
            .unwrap_or_default()
    }

    /// Submit a decision for `request_id`. Returns `Ok(())` when a
    /// matching waiter was unblocked; `Err` for unknown ids or
    /// already-answered entries.
    pub fn submit(
        &self,
        session_id: &str,
        request_id: &str,
        decision: ApprovalDecision,
    ) -> Result<(), String> {
        let mut guard = self.inner.lock().expect("ApprovalQueue lock poisoned");
        let entry = guard
            .get_mut(session_id)
            .ok_or_else(|| format!("no pending approvals for session {session_id}"))?;
        let tx = entry
            .waiters
            .remove(request_id)
            .ok_or_else(|| format!("approval {request_id} already answered or expired"))?;
        entry.pending.retain(|p| p.request_id != request_id);
        // Drop empty session entries to avoid leaking keys forever.
        if entry.pending.is_empty() && entry.waiters.is_empty() {
            guard.remove(session_id);
        }
        let _ = tx.send(decision);
        Ok(())
    }

    /// Drop a request after a timeout. Mirrors `submit` but never
    /// notifies the waiter — the forwarding handler's
    /// `tokio::time::timeout` already fell through, so the receiver
    /// half is gone.
    pub fn timeout(&self, session_id: &str, request_id: &str) {
        let mut guard = self.inner.lock().expect("ApprovalQueue lock poisoned");
        if let Some(entry) = guard.get_mut(session_id) {
            entry.pending.retain(|p| p.request_id != request_id);
            entry.waiters.remove(request_id);
            if entry.pending.is_empty() && entry.waiters.is_empty() {
                guard.remove(session_id);
            }
        }
    }
}

//! `ApprovalHandler` impl that bridges the helper's AgentLoop into
//! the [`ApprovalQueue`] so the hub can answer approval prompts
//! out-of-band via the polling tools.

use std::sync::Arc;
use std::time::Duration;

use async_trait::async_trait;
use serde_json::{json, Value};
use tokio::sync::mpsc;
use zunel_core::{ApprovalDecision, ApprovalHandler, ApprovalRequest};

use crate::commands::mcp::approval_queue::ApprovalQueue;

pub struct ForwardingApprovalHandler {
    pub queue: Arc<ApprovalQueue>,
    pub session_id: String,
    pub timeout: Duration,
    /// Optional progress sink — populated when the helper is
    /// streaming. We emit a hint so a streaming hub can react to
    /// the queued approval without waiting for its next poll.
    pub progress: Option<mpsc::Sender<Value>>,
    /// Echoed in the progress hint so the hub knows which token's
    /// stream the hint belongs to.
    pub progress_token: Option<Value>,
}

#[async_trait]
impl ApprovalHandler for ForwardingApprovalHandler {
    async fn request(&self, req: ApprovalRequest) -> ApprovalDecision {
        let (request_id, rx) = match self.queue.enqueue(&self.session_id, req.clone()) {
            Ok(pair) => pair,
            Err(err) => {
                tracing::warn!(session_id = %self.session_id, %err, "approval enqueue failed");
                return ApprovalDecision::Deny;
            }
        };
        if let (Some(sink), Some(token)) = (self.progress.as_ref(), self.progress_token.as_ref()) {
            let _ = sink
                .send(json!({
                    "jsonrpc": "2.0",
                    "method": "notifications/progress",
                    "params": {
                        "progressToken": token,
                        "message": format!(
                            "[awaiting approval] tool={} request_id={}",
                            req.tool_name, request_id
                        )
                    }
                }))
                .await;
        }
        match tokio::time::timeout(self.timeout, rx).await {
            Ok(Ok(decision)) => decision,
            Ok(Err(_)) => {
                self.queue.timeout(&self.session_id, &request_id);
                ApprovalDecision::Deny
            }
            Err(_) => {
                self.queue.timeout(&self.session_id, &request_id);
                ApprovalDecision::Deny
            }
        }
    }
}

//! `helper_approve` MCP tool — submit an approve/deny decision
//! against a queued approval request. Registered only when
//! `--mode2-approval forward` is set.

use std::sync::Arc;

use async_trait::async_trait;
use serde_json::{json, Value};
use zunel_core::ApprovalDecision;
use zunel_tools::{Tool, ToolContext, ToolResult};

use crate::commands::mcp::approval_queue::ApprovalQueue;

pub struct HelperApproveTool {
    pub queue: Arc<ApprovalQueue>,
}

#[async_trait]
impl Tool for HelperApproveTool {
    fn name(&self) -> &'static str { "helper_approve" }
    fn description(&self) -> &'static str {
        "Approve or deny a queued approval request inside a helper session. \
         Use after `helper_pending_approvals` to act on its output."
    }
    fn parameters(&self) -> Value {
        json!({
            "type": "object",
            "required": ["session_id", "request_id", "decision"],
            "properties": {
                "session_id": {
                    "type": "string",
                    "description": "Full helper session id."
                },
                "request_id": {
                    "type": "string",
                    "description": "request_id from `helper_pending_approvals`."
                },
                "decision": {
                    "type": "string",
                    "enum": ["approve", "deny"],
                    "description": "Forwarded to the helper's blocked approval handler."
                }
            }
        })
    }
    fn concurrency_safe(&self) -> bool { false }
    async fn execute(&self, args: Value, _ctx: &ToolContext) -> ToolResult {
        let Some(session_id) = args.get("session_id").and_then(Value::as_str) else {
            return ToolResult::err("helper_approve: missing string `session_id`");
        };
        let Some(request_id) = args.get("request_id").and_then(Value::as_str) else {
            return ToolResult::err("helper_approve: missing string `request_id`");
        };
        let Some(decision_str) = args.get("decision").and_then(Value::as_str) else {
            return ToolResult::err("helper_approve: missing string `decision`");
        };
        let decision = match decision_str {
            "approve" => ApprovalDecision::Approve,
            "deny" => ApprovalDecision::Deny,
            other => {
                return ToolResult::err(format!(
                    "helper_approve: unknown decision {other:?}; expected 'approve' or 'deny'"
                ))
            }
        };
        match self.queue.submit(session_id, request_id, decision) {
            Ok(()) => ToolResult::ok(format!(
                "decision recorded: {decision_str} for {request_id}"
            ))
            .with_meta(json!({"ok": true})),
            Err(err) => ToolResult::err(format!("helper_approve: {err}")),
        }
    }
}
